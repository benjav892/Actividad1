Estrategia de Ramificación: Trunk-Based Development (TBD)
Implementamos Trunk based development por que:
-facilita una integracion continua.
-permite una entrega más rápida. 
-reduce la complejidad de conflictos-

convenciones del proyecto

Seguiremos el estandar convencional:
feature/... para cosas nuevas.
hotfix/... para urgencias.
bugfix/... para errores normales.

Flujo de trabajo: Todo cambio al main debe pasar por un pull request y su revision.
